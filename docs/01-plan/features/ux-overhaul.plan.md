# Plan: UX Overhaul - 사용성 전면 개선

## 개요

현재 프론트엔드의 핵심 UX 문제점들을 분석하고 해결하는 계획.
사용자 피드백: "UX 고려 안 했어? 프론트가 너무 구린데? 사용성 생각 하나도 안 함?"

---

## 문제 분석 (5가지 핵심 이슈)

### Issue 1: 제안(Proposal) 생성 방법 없음
- **현상**: "투표하기" 페이지에 들어가면 "Deploy New Poll" 버튼이 있지만, 제안 제목/설명/기간을 입력하는 폼이 없음
- **문제**: 하드코딩된 `'Demo Poll'` + `3600초`로만 생성됨. 사용자가 실제 안건을 만들 수 없음
- **영향**: 투표 시스템의 핵심 기능 부재

### Issue 2: 일반/이차 투표 모드 분리의 혼란
- **현상**: "일반 투표" / "이차 투표" 탭이 페이지 상단에 나란히 있음
- **문제**: 일반 사용자는 이 차이를 모름. 투표 모드는 **제안 생성 시 결정**되어야 하는데, 투표자가 선택하는 것처럼 보임
- **영향**: 한 제안에 일반/이차를 섞어 투표하면 안 됨 - UI가 이를 허용하는 것처럼 오해

### Issue 3: Phase 프로세스 바와 버튼의 시각적 혼동
- **현상**: Phase Bar (투표하기 | 데이터 병합 | 투표 처리 | 결과 확정)가 파란색/검은색 버튼과 동일한 스타일
- **문제**: 진행 상황 표시인데 클릭 가능한 버튼으로 보임. 모드 선택기(일반/이차)와도 시각적 구분 안 됨
- **영향**: 사용자가 Phase를 클릭하려 함 / 어디가 버튼이고 어디가 상태 표시인지 혼란

### Issue 4: 단계별 흐름이 한 페이지에 나열
- **현상**: 등록 → 투표 생성 → 투표 → 키 관리가 모두 하나의 긴 스크롤 페이지
- **문제**: 처음 방문한 사용자가 어디서부터 봐야 할지 모름. 완료된 단계도 계속 보임
- **영향**: 인지 부하 높음, 현재 해야 할 일이 불명확

### Issue 5: 랜딩 페이지의 관련 없는 콘텐츠
- **현상**: "금권정치 vs. 공정성" 비교 테이블이 아직 남아있음
- **문제**: MACI만 남겼는데 구 D1/D2 설명이 랜딩에 존재. 사용자 흐름과 무관한 정보
- **영향**: 실제 투표 흐름(등록 → 투표 → 결과)에 대한 안내 부족

## 해결 방안

### Fix 1: 제안 생성 폼 추가
```
현재: handleDeployPoll() → 하드코딩 'Demo Poll', 3600초
변경: 제안 생성 폼 → 제목, 설명, 기간(시간), 투표 모드(D1/D2) 입력
```
- 새 컴포넌트: `CreatePollForm.tsx`
- 필드: 제목(필수), 설명(선택), 기간(1~72시간 슬라이더), 투표 모드(기본: 일반)
- 생성 후 → 자동으로 해당 투표 페이지로 이동

### Fix 2: 투표 모드 선택기를 제안 생성 시로 이동
```
현재: 투표자가 투표 페이지에서 D1/D2 선택
변경: 제안 생성자가 생성 시 모드 선택. 투표자는 해당 모드로만 투표
```
- `votingMode`를 MACIVotingDemo에서 제거
- CreatePollForm에 모드 선택 통합
- Poll 생성 시 모드 정보를 localStorage에 저장
- 투표 폼은 해당 Poll의 모드를 읽어서 자동 설정

### Fix 3: Phase 바를 스텝퍼(Stepper)로 재설계
```
현재: [투표하기|데이터 병합|투표 처리|결과 확정] ← 버튼처럼 보임
변경: ① 등록 ─── ② 투표 ─── ③ 처리 중 ─── ④ 결과
      ✓ done     ● 현재      ○ 대기       ○ 대기
```
- 원형 번호 + 연결선 디자인 (프로그레스 스텝퍼)
- 완료: 체크마크 + 녹색, 현재: 파란 펄스, 대기: 회색
- 클릭 불가 시각적 표현 (cursor: default, 호버 효과 없음)
- 현재 단계의 내용만 카드로 표시 (나머지는 접힘)

### Fix 4: 단계별 카드 UI (아코디언/위자드)
```
현재: 1.등록 2.투표생성 3.투표 4.키관리 → 모두 펼쳐져 있음
변경: 현재 단계만 활성화. 완료 단계는 접혀서 요약만 표시
```
- 등록 완료 → "✓ 등록 완료 (0xAbCd...)" 한 줄로 접힘
- 활성 단계만 큰 카드로 표시
- 키 관리는 투표 단계 안의 고급 옵션(접힌 상태)으로 이동

### Fix 5: 랜딩 페이지 콘텐츠 MACI 중심으로 교체
```
현재: 금권정치 vs 공정성 비교 테이블 (구 D1/D2)
변경: MACI 강점 중심의 콘텐츠
```
- "금권정치 vs 공정성" 섹션 → "왜 MACI인가?" 섹션으로 교체
  - 강압 방지: 키 변경으로 이전 투표 무효화
  - 완전한 프라이버시: 암호화된 투표, 아무도 볼 수 없음
  - 검증 가능: 온체인 영지식 증명
- CTA의 "지갑 연결 → 투표 등록 → 투표" 흐름 유지

## 구현 순서

1. **스텝퍼 + 아코디언 UI** — Phase 바 재설계 + 단계별 접힘/펼침
2. **제안 생성 폼** — CreatePollForm 컴포넌트 + 투표 모드 선택 통합
3. **투표 모드 선택기 제거** — MACIVotingDemo에서 상단 모드 탭 제거
4. **KeyManager 접힌 상태** — 고급 옵션으로 이동
5. **랜딩 페이지** — QV 비교 → MACI 강점 섹션
6. **i18n 업데이트** — 새/변경 번역 키
7. **CSS 정리** — 통일된 시각적 계층구조
